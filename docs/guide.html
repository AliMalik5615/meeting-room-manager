Purpose

The Meeting Room Manager is a PowerShell-based provisioning tool that enables service desk staff to create, modify, delete, and audit Exchange Online meeting room mailboxes without navigating the Exchange Admin Centre manually.

Room configuration data is supplied via a structured Excel workbook. The tool validates the workbook, previews all planned actions before execution, and requires typed confirmation before any change is applied to Exchange Online. A colour-coded audit check is also available to verify that live room configurations match the expected values in the workbook.

Prerequisites

Approval required. A ticket containing manager or physical assets approval must be present before any create, modify, or delete operation is performed.

Tool requirements

  • Windows PowerShell 5.1 (Desktop edition) — the script will not run under PowerShell 7 or pwsh
  • Exchange Administrator role must be active before launching the tool (JIT / PIM activation where applicable)
  • Microsoft Excel desktop must be installed on the workstation — the tool uses Excel COM automation to read the workbook
  • The tool is STA-dependent; it must be run on a standard managed desktop. Thin clients and non-standard endpoints are not supported
  • For Modify operations: the Read and Manage distribution group listed in the workbook must already exist in Exchange Online before running. The tool does not create distribution groups automatically

Workbook requirements

  • File extension must be .xlsx or .xls
  • Workbook name can be anything
  • Worksheet (tab) name must be exactly Add New Meeting Rooms
  • Header names must not be changed — column order does not matter
  • Extra columns are permitted and will be ignored by the tool

Launching the tool

Navigate to the script folder and copy it to your local desktop or C:\ drive. The tool can be launched in two ways depending on your environment.

Launching via PowerShell

  1. 1

    Open Windows PowerShell 5.1 — search for Windows PowerShell in the Start menu. Do not use PowerShell 7 or the PowerShell ISE.

  2. 2

    Run the following command to allow the script to execute in this session. This applies to the current PowerShell window only and does not change any system-wide policy.

    Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
  3. 3

    Run the script by typing its full file path. Replace the path below with the actual location of the .ps1 file on your machine.

    # Example — adjust path to match your copy of the script
    & "C:\Users\YourName\Desktop\Meeting Room Manager v2.3\Meeting Room Manager Tool.ps1"
    Tip: In File Explorer, hold Shift and right-click the .ps1 file, then select Copy as path. Paste that directly into PowerShell after the & operator to avoid typing errors.

On launch, the tool will check for the ExchangeOnlineManagement module, install it if absent, and prompt for Exchange Online authentication. Sign in with the account that has the Exchange Administrator role active.

1

Create meeting room(s)

Creates one or more room mailboxes in Exchange Online using the data supplied in the workbook. Create performs a thin provisioning — it establishes the mailbox identity only. Full configuration (location, booking policy, room list membership, delegates) is applied in the Modify step.

  1. 1

    Launch the tool. Sign in to Exchange Online when prompted.

  2. 2

    At the Select operation menu, type 1 and press Enter. A file picker dialog will open — browse to and select the completed workbook file.

  3. 3

    The tool parses the workbook and displays a Parsed row summary followed by a Planned action summary. Review both before proceeding. Rows with validation errors are shown as Blocked and will not be processed.

    When ready, type CREATE exactly and press Enter to confirm. Type X to return to the main menu without making any changes.

    No changes are made until you type CREATE. The summary screen is a dry run — review it carefully before confirming.
  4. 4

    The tool processes each room row and displays live status as it runs. On completion, an Execution complete [Create] summary is shown with a count of completed, skipped, and blocked rows. Press any key to return to the main menu.

  • 5

    Optionally, verify the result in the Exchange Admin Centre under Recipients → Resources. The room mailboxes will appear in the list with their display name and email address confirmed. Location and contact fields will be empty at this stage — this is expected. Full configuration is applied in the next step.

    • After Create: run Modify (Option 2) to apply the full room configuration — location details, booking policy, room list membership, and delegate groups are not set during the Create step.
    2

    Modify meeting room(s)

    Applies the full configuration to existing room mailboxes — including location data, booking policy settings, room list membership, approval workflows, and delegate groups. This is typically run immediately after Create, but can also be used to update configuration on rooms that already exist in Exchange Online.

    Before running Modify: confirm that the Read and Manage distribution group listed in the workbook already exists in Exchange Online. The tool does not create DLs automatically — see your team's manual guide for DL creation steps.
    1. 1

      Launch the tool (or return to the main menu if already open). Sign in if prompted.

    2. 2

      Type 2 and press Enter. Select the same workbook used during Create. The workbook must contain the full schema including all location, policy, and group fields.

    3. 3

      Review the parsed row summary and planned action summary. When satisfied, type MODIFY exactly and press Enter.

    4. 4

      The tool applies each configuration property per room and displays progress. Any warnings (such as missing calendar permissions on a property) are shown in yellow but do not stop execution. Review the completion summary and press any key to return.

    5. 5

      Optionally, verify the location and contact details in the Exchange Admin Centre under Recipients → Resources. Select a room and open Manage resource contact information — the Street, City, State/Province, and ZIP/Postal code fields should now be populated with the values from the workbook.

    6. 6

      Still in the Exchange Admin Centre, open Manage delegates for the same room. The Read and Manage security group should appear as a listed delegate alongside NT AUTHORITY\SELF, confirming that mailbox permissions were assigned successfully.

    Tip: After Modify, run Check (Option 4) to verify that the live Exchange Online configuration matches the workbook values.
    3

    Delete meeting room(s)

    Permanently removes room mailboxes from Exchange Online. Delete supports two input methods: a workbook (to delete multiple rooms in one run) or a manually typed SMTP address (to delete a single room without a workbook).

    This operation is irreversible. Deleted room mailboxes cannot be recovered through this tool. Confirm approval is documented in the ticket before proceeding.
    1. 1

      Launch the tool (or return to the main menu if already open).

    2. 2

      Type 3 and press Enter. A file picker will open — select the workbook, or close the picker and type the resource email address manually to delete a single room without a workbook.

    3. 3

      The tool shows the parsed row summary and a delete preview including the number of rooms targeted. Type DELETE exactly and press Enter to proceed.

    4. 4

      For each room, the tool prompts for a final confirmation by requiring the exact SMTP address to be typed or pasted. This prevents accidental deletion — if the address does not match, the operation fails for that row. Type X at any individual room prompt to skip that room and continue with the remaining rows.

      The SMTP address can be copied from the PowerShell window and pasted at the prompt to reduce the risk of a typing error.
    4

    Check meeting room(s)

    Fetches the live configuration of one or more rooms from Exchange Online and compares each value against the expected values in the workbook. Results are colour-coded for quick auditing. The check output can be exported as a .txt file for attaching to the service request as evidence.

    Output colour coding

    Green — exact match between expected and actual values
    Yellow — display name to SMTP match (partial match)
    Red — mismatch between expected and actual values
    1. 1

      Launch the tool (or return to the main menu if already open).

    2. 2

      Type 4 and press Enter. Select the workbook using the file picker, or close the picker and type a single resource email address to check one room without a workbook.

    3. 3

      The tool queries Exchange Online for each room and displays the Expected vs Actual comparison with colour coding. Review each field — red rows indicate a configuration mismatch that may require a follow-up Modify run. When checking multiple rooms in a single run, each room is displayed as a separate block identified by its row number and SMTP address.

      The three screenshots below show a batch check across three rooms with different configurations — varying floor, capacity, tags, and approval settings — all returning fully green results after a successful Modify run.

      Yellow on Booking Delegate Group is expected when Requires Approval = No. When a room does not require approval, no delegate group is assigned — the tool correctly displays this field as yellow (partial match) rather than red (mismatch). This is not an error and does not require remediation.
    4. 4

      At the end of the check output, type Y to export the results as a .txt file, or press any other key to return to the main menu without exporting. The tool confirms how many result files were generated and displays the full path to each one.

      Exported results are saved to the results\ folder inside the tool's root directory — one file per room checked, named by SMTP address. Each file contains the full business view, booking delegate settings, and optional technical detail for that room as fetched live from Exchange Online. Copy the contents of the relevant file and paste it into the service request as evidence of the room configuration.

    Use Check as a post-provisioning audit step. Run it after every Create + Modify cycle and attach the exported results to the service request before closing the ticket.
